Deploying Symantec Digital Notarization Client software

The MSI packaging for the Symantec Digital Notarization Client allows a Windows 2000 domain administrator to remotely deploy the Notarization Client to users within his domain by leveraging Microsoft's Windows Installer technology and active directory based group policies.

The Symantec Digital Notarization Client MSI packages

Go Secure! for Web Applications contains the following MSI packages for deploying Symantec Notarization Client.

  • Symantec Digital Notarization Client and PTA for IE
    • vsnzpta.msi - the Digital Notarization Client and PTA MSI package. 
    • vsnzpta.mst - This is a transform file which customize the installation of the Notarization Client for a specific account (customer). The mst file is generated by the administrator using the tools that are packaged along with the MSI file. 
  • Symantec Digital Notarization Client for IE
    • vsdnotz.msi - the Digital Notarization Client.
    • vsdnotz.mst - This is a transform file which customize the installation of the Notarization Client for a specific account (customer). The mst file is generated by the administrator using the tools that are packaged along with the MSI file.

Note: The Digital Notarization Client requires the PTA for its operation. Hence, we recommend that you use the vsdnotz.msi/mst packages only if you have already deployed the PTA using the Assign to machine approach. We recommend that you use the vsnzpta.msi/mst packages.

  • Symantec Digital Notarization Client for Netscape 4.x
    • nppta3nz.msi - the Digital Notarization Client and PTA MSI package for Netscape 4.x browsers. 
    • nppta3nz.mst - This is a transform file which customize the installation of the Notarization Client for a specific account (customer). The mst file is generated by the administrator using the tools that are packaged along with the MSI file.
  • Symantec Digital Notarization Client for Netscape 6.x
    • n6pta3nz.msi - the Digital Notarization Client and PTA MSI package for Netscape 6.x browsers. 
    • n6pta3nz.mst - This is a transform file which customize the installation of the Notarization Client for a specific account (customer). The mst file is generated by the administrator using the tools that are packaged along with the MSI file. 

These packages can be downloaded as part of the following zip files.

  1. dnzmsikit.zip- This contains the MSI files and the tools required to generate the mst file. When expanded the following directories are obtained.
    • engine
    • templates
    • fragments

Generating the Notarization mst packages

The mst packages allows an administrator to customize his deployment of the Notarization Client. In order to generate the MST pacakages the administrator needs a policy file. The policy file is obtained by running the policy wizard in the Managed PKI Control Center. Once the policy file is obtained the administrator needs to do the following :

  • You need to generate the Notarization mst packages on a Windows 2000 machine.
  • Navigate to the engine directory (obtained by unzipping the dnzmsikit.zip file)
  • Run the dnzcustomize.bat file with the following parameters
dnzcustomize.bat <dest_dir> <policy_file>, where
    • <dest_dir> is the destination directory where the mst files will be generated, and
    • <policy_file> is the name and location of the policy file that will be used while performing the customization.
  • After running the batch file, the various msi and mst files will be available in the specified destination directory.

Publishing the Notarization Client packages to a user

Following steps outline the process for publishing the msi packages.

  1. Right click on the OU in the active directory to whose users the software is to be published.
  2. Select Properties
  3. Select the Group Policy tab
  4. Select the Group Policy that applies to the OU if one exists, or create a new one if it does not.
  5. Click on Edit.
  6. This will bring up another window that specifies the group policies.
  7. Select User Configuration and expand the tree corresponding to this selection.
  8. In the expanded tree right click on Software Installation and select New Package
  9. Select the MSI file that contains the software that should be published.
  10. A dialog box will appear asking if the package should be published or assigned. Select "Advanced published or Assigned".
  11. Another dialog box appears. Select the "Deployment" Tab. Choose the "Publish" Radio button.
  12. Select the "Modifications" Tab.
  13. Click on the Add button and in the file selection dialog box that appears select the mst file.
  14. Click OK to complete the procedure. Assigning the pta packages to a machine

Assigning the Notarization packages to a machine

Following steps outline the process for assigning the msi packages to a machine

  1. Right click on the OU in the active directory to whose machines the software is to be published.
  2. Select Properties
  3. Select the Group Policy tab
  4. Select the Group Policy that applies to the OU if one exists, or create a new one if it does not.
  5. Click on Edit.
  6. This will bring up another window that specifies the group policies.
  7. Select Machine Configuration and expand the tree corresponding to this selection.
  8. In the expanded tree right click on Software Installation and select New Package
  9. Select the MSI file that contains the software that should be published.
  10. A dialog box will appear asking if the package should be published or assigned. Select assigned.
  11. Another dialog box appears. Select the "Deployment" Tab. Choose the "Assign" Radio button.
  12. Select the "Modifications" Tab.
  13. Click on the Add button and in the file selection dialog box that appears select the mst file.
  14. Click OK to complete the procedure.
