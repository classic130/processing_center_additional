Security Properties

The Security Properties dialog box lets you select the algorithm used for encryption and hashing. It also allows you to set a screen saver.

To change Security properties:

  1. Open the Safe Properties.
  2. Click the Advanced tab and then the Security button.
  3. Change any settings and then click OK. The settings are described below.

Encryption Algorithm -
This option determines which algorithm the File Encryption Tool uses when encrypting files. Your Administrator might have selected an algorithm for you. If you are unfamiliar with the algorithms, accept the default setting. The algorithms are:

  • RC2-40bits
  • RC2-56bits
  • RC2-128bits
  • RC4
  • DES
  • 3DES

Hashing Algorithm -
This option sets the Hashing Algorithm used in the File Encryption Tool for signing files. If the Safe is not configured to sign files, this option is disabled. The hash algorithms are SHA1, MD4, and MD5.

Lock Desktop -
These options show you any screen savers which are available on your machine. Any changes you make here are immediately reflected in the Windows screen saver and vice-versa. You can use these options to set up a password-protected screen saver that locks your desktop when you stop working for a set amount of time.

If you leave the File Encryption Tool running on your computer, it is possible for someone to use the Safe when you leave your computer. By setting up a screen saver that activates itself in a short amount of time (for example, two minutes), you can prevent others from using your computer while you are away. For more information on setting up the screen saver, see the Windows online help.

Securely delete the original file after encryption -
Use this option to have the File Encryption Tool completely delete any trace of the original file from your computer. Some Windows applications can create temporary files or backups of your files. The Safe has no way of knowing these files exist, so you need to make sure that you encrypt any copies or backups as well as the original file. See the documentation that came with your applications for information on what files, if any, the application creates.

When you move a file to the Safe or encrypt it in place, the original file is deleted from you hard disk. However, because of the way Windows normally deletes files, it is possible for someone who has access to your computer to use highly sophisticated software to read all or parts of the file on your hard disk.

When an application (such as the Safe) deletes a file, Windows leaves the file contents on the hard disk in a state where they can be overwritten by other files. Until another file (or set of files) is saved to the hard disk, the deleted file remains on the hard disk. You can think of it as a chalk board that is erased only when you want to write on it; until you write something new, you can still read the old writing.

This secure deletion option completely erases the file so that it cannot be read at all by any software or hardware. This option is on (checked) by default.
